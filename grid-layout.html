HTML Elements Overview

Paragraphs (<p>)

Used for blocks of text, paragraphs are the foundation of readable content.

Read More
Paragraphs are block-level elements that automatically add spacing above and below. They help structure text for clarity and accessibility.

Images (<img>)

Embed visual content such as photos, icons, or diagrams.

Read More
The <img> tag is used to display images. Always include alt text for accessibility and SEO.

Links (<a>)

Allow users to navigate between pages or external sites.

Read More
The <a> tag creates hyperlinks. Use descriptive link text for usability and accessibility.

Lists (<ul>, <ol>)

Organize items in bullet or numbered lists for clarity.

Read More
Use unordered lists (<ul>) for bullets and ordered lists (<ol>) for numbered steps. Each item goes in a <li>.

Tables (<table>)

Display tabular data in rows and columns.

Read More
Tables are best for data, not for layout. Use <tr> for rows and <td> for cells.

Forms (<form>)

Collect user input with fields, checkboxes, and buttons.

Read More
Forms use <input>, <textarea>, <select>, and <button> to gather data from users.

Headings (<h1>–<h3>)

Define content hierarchy and improve accessibility.

Read More
Use headings to structure your page. <h1> is for the main title, <h2> for sections, and <h3> for subsections.

Containers (<div>, <section>)

Group and structure content for layout and styling.

Read More
<div> is a generic container. <section> is for thematically grouped content.

Headers, Footers, Navigation (<header>, <footer>, <nav>)

Define top, bottom, and navigation areas of your site.

Read More
<header> is for site or section headers, <footer> for closing info, and <nav> for navigation menus.

Sidebars (<aside>)

Complementary content such as menus, ads, or widgets.

Read More
<aside> is used for content related to the main content, like sidebars or callouts.

Layout Techniques Showcase

HTML Tables

Uses <table> for page structure. Each section is placed in a table cell.

Learn More
How it works:
  • Arrange layout using rows and columns.
Pros:
  • Easy for tabular data
  • Simple to understand
Cons:
  • Not semantic for layout
  • Not responsive
  • Hard to maintain

CSS Float

Uses float to position elements side by side.

Learn More
How it works:
  • Floats columns left or right for layout.
Pros:
  • Simple, widely supported
  • Easy for basic layouts
Cons:
  • Can cause clearfix issues
  • Less flexible
  • Not ideal for responsive or complex layouts

CSS Flexbox

Uses display: flex for flexible, responsive layouts in one dimension.

Learn More
How it works:
  • Aligns and distributes space among items in a container.
Pros:
  • Responsive
  • Easy alignment
  • Great for 1D layouts
Cons:
  • Not ideal for complex 2D layouts

CSS Frameworks

Uses frameworks like Bootstrap for responsive, ready-made grids and interactivity.

Learn More
How it works:
  • Prebuilt grid and UI components for fast development.
Pros:
  • Fast, responsive, interactive
  • Lots of features
  • Cross-browser support
Cons:
  • Adds extra code
  • Less custom
  • Learning curve for framework classes

CSS Grid

Uses display: grid for powerful, responsive 2D layouts.

Learn More
How it works:
  • Defines rows and columns for complex layouts.
Pros:
  • Best for 2D layouts
  • Very flexible and responsive
  • Semantic
Cons:
  • Newer, may be overkill for simple pages
  • Browser support needed for older browsers
