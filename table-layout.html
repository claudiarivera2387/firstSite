HTML Table-Based Layout

Learn how HTML tables can be used to structure web page layouts, their strengths and weaknesses, and where they still matter today.

What Are HTML Tables?

HTML tables use the <table> element to organize content into rows (<tr>) and columns (<td> or <th>). In early web design, tables were often used to create page layouts by placing navigation, content, and sidebars into different table cells.

<table>
  <tr><th colspan="2">Header</th></tr>
  <tr><td>Navigation</td><td>Main Content</td></tr>
  <tr><td colspan="2">Footer</td></tr>
</table>

Pros

  • Simple to understand and implement
  • Consistent structure across browsers
  • Good for tabular data

Cons

  • Not semantic for layout (bad for accessibility & SEO)
  • Hard to make responsive for mobile devices
  • Difficult to maintain and update
  • Slower page loads due to complex markup

Use Cases Today

  • Emails: Many email clients only support table-based layouts for consistent rendering.
  • Tabular Data: Displaying spreadsheets, schedules, or reports.
  • Legacy Applications: Older intranet sites or admin panels may still use table layouts.

Tip: For modern web design, use CSS Grid or Flexbox for layout, and reserve tables for actual tabular data.
